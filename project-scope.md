## Problem

I am getting a lot of issue tickets from my students via email. My agents need a lot of time going through all of them and most of them require just generic answers. We have a system for that but it sounds very generic and people don't like getting these kind of generic answers.

## Solution

Build an application that will bqe triggered whenever I receive a support email and automatically ask Ai to go through student's question and provide a nice and personalized answer and if it can't do that, send it further to one of our human agents.

## Ticket Statuses

- **Open** — ticket is active and awaiting a response
- **Resolved** — ticket has been answered (by AI or agent)
- **Closed** — ticket is no longer active

## Ticket Categories

- General question
- Technical question
- Refund request

## User Roles

- **Admin** — deployed with the system; can create and manage agents
- **Agent** — created by admin; handles escalated tickets

## Features

- Receive support emails and create tickets
- Auto-generate human-friendly responses using a knowledge base
- Ticket list with sorting and filtering
- AI-powered ticket classification
- AI summaries
- AI-suggested replies
- User management (admin only)
- Dashboard to view and manage all tickets
